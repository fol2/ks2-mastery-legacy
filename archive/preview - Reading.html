Optional AI

In a single local HTML file, any direct API key use is client-side. Session-only storage is the safer default. Some providers may still reject direct browser calls because of CORS or auth rules. The core app works fully with AI turned off, and the provider code is isolated so you can later attach a tiny proxy.

How this app is designed

The loop is simple on purpose: read a real passage, answer without answer leakage, get focused feedback after effort, then see weak skills return later in mixed review. Smart review weights weak content domains, overdue review, under-practised passage types and recent misses so problems reappear before they fade.

Why the feedback waits

No evidence is highlighted before the attempt The skill label is hidden unless scaffold mode is on A first wrong answer gets a nudge, not the answer Test mode delays all feedback to the end
Built-in content is original and SATs-aligned in style, not copied from copyrighted test materials. AI can add fresh original passages, but the core loop, marking, analytics and review all work offline.
